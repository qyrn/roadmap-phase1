5 challenges pour valider les concepts du module réseau

01

Service Discovery

concept : ss / netstat — connexions actives

Le serveur écoute sur un port non-standard. Identifie-le avec les bons outils réseau, puis récupère le flag via l'endpoint ci-dessous.

GET http://localhost:7373/flag/1
Indice
ss -tulpn | grep node
netstat -tulpn | grep node
02

Header Hunt

concept : HTTP headers — inspection de réponse

Le flag de ce challenge n'est pas dans le body de la réponse. Il est dans un header HTTP custom. Inspecte les headers complets.

GET http://localhost:7373/flag/2
Indice
curl -I http://localhost:7373/flag/2
curl -v http://localhost:7373/flag/2
03

Method Matters

concept : méthodes HTTP — GET vs POST

Cet endpoint refuse les requêtes GET. Envoie la bonne méthode HTTP avec le body JSON attendu pour obtenir le flag.

POST http://localhost:7373/flag/3
Body: {"method": "POST"}
Indice
curl -X POST -H "Content-Type: application/json" \
-d '{"method":"POST"}' http://localhost:7373/flag/3
04

Virtual Host

concept : DNS — résolution de noms, Host header

Ce flag n'est accessible qu'au bon hostname. DNS traduit les noms en IP — ici, simule cette résolution avec le bon header HTTP.

GET http://localhost:7373/flag/4
Indice
curl -H "Host: lab2.local" http://localhost:7373/flag/4
05

PIN Bruteforce

concept : brute force — HTTP POST, automatisation

Un formulaire est protégé par un code PIN à 4 chiffres (0000 → 9999). Écris un script bash ou Python pour tester toutes les combinaisons et récupérer le flag.

POST http://localhost:7373/flag/5
Body: {"pin": "XXXX"}
Indice
# Bash
for i in $(seq -w 0000 9999); do
  r=$(curl -s -X POST -H "Content-Type: application/json" \
       -d "{\"pin\":\"$i\"}" http://localhost:7373/flag/5)
  echo "$r" | grep -q "flag" && echo "$r" && break
done
